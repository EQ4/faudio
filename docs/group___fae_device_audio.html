Detailed Description

Real-time audio devices.

These device run processors on the input and output the underlying system, typically physical audio interfaces. A running audio computation is represented by a stream. Access to the current device setups is provided by sessions.

Sessions implement
  • fae_equal_t
  • fae_destroy_t
  • fae_string_show_t
Devices implement
  • fae_equal_t
  • fae_string_show_t

Devices implement Equal and Show, but now destroy, as they are always managed by the underlying session.

Streams implement
  • fae_time_clock_t
  • fae_message_sender_t
  • fae_message_receiver_t
  • fae_destroy_t
  • fae_string_show_t

Stream implements Clock by counting the number of processed samples. Precision is usually limited to the current frame size.

Stream also implements SenderInterface and ReceiverInterface, dispatching messages to the underlying processor network. The address of a processor can be obtained by fae_processor_address.

fae_ptr_t  error_data

Begin a new session, and retain it for the duration of a call to the given function.

The given function will be called once after the session has created. The session will be ended after the callback function has returned. If an error occurs while starting the session, the error callback is invoked in place of the session callback.
